# 💳 Система управления банковскими картами

Backend-приложение для управления банковскими картами с ролевой моделью (ADMIN/USER), шифрованием данных, переводами между своими картами и полной Docker-поддержкой.

## 📋 О проекте

Реализована система, где:
- **Администратор** может создавать, блокировать, активировать и удалять любые карты, управлять пользователями.
- **Пользователь** может просматривать свои карты (с пагинацией и поиском), запрашивать блокировку, переводить средства между своими картами и смотреть баланс.

| Категория | Технологии |
|-----------|-------------|
| **Язык** | Java 21 |

| **База данных** | PostgreSQL 16 |

| **Документация API** | Swagger / OpenAPI |
| **Контейнеризация** | Docker, Docker Compose |

## 🔐 Основные возможности

### 👑 Администратор
- ✅ Управление пользователями (создание, удаление)
- ✅ CRUD для карт всех пользователей
- ✅ Блокировка / активация / удаление карт
- ✅ Просмотр всех карт и транзакций

### 👤 Пользователь
- ✅ Просмотр своих карт (пагинация + поиск по статусу/владельцу)
- ✅ Переводы между **своими** картами
- ✅ Запрос на блокировку карты
- ✅ Просмотр баланса
- ✅ Безопасное отображение номера карты: `**** **** **** 1234`

## 🛡️ Безопасность
- JWT-аутентификация (Access Token)
- Пароли пользователей хешируются (BCrypt)
- Номера карт шифруются в БД
- Ролевая модель: `ADMIN`, `USER`
- Маскирование номеров карт в API-ответах

## 📦 Запуск проекта

### Требования
- Docker & Docker Compose
- Port 8080, 5432, 5050 свободны
> **⚠️ Важное примечание для проверяющего (тестовая среда)**

Для упрощения процесса развертывания и тестирования проекта **все секреты, пароли и ключи** оставлены открытыми в файле `docker-compose.yml` и `application-docker.yml`.

**Что содержится открыто:**
- Пароль БД
- JWT Secret (Base64)
- Соль и пароль шифрования карт
- Логин/пароль администратора

**Почему так сделано:**
- ✅ Ускорение процесса развертывания (не нужно создавать `.env` файлы)
- ✅ Упрощение тестирования для ревьюера
- ✅ Проект находится в стадии демонстрации, не предназначен для production

**В реальном проекте** все эти значения были бы вынесены в:
- Переменные окружения (`.env` файл, не добавляемый в Git)
- Docker secrets
- Vault (HashiCorp Vault, AWS Secrets Manager и т.д.)
